Installation

  1. Download the latest version from Modrinth
  2. Place the .jar into your plugins/ folder
  3. Make sure LuckPerms is installed
  4. Start or restart your server
  5. LuckTAB generates plugins/LuckTAB/config.yml
  6. Edit the config and run /lucktab reload

Quick Setup

Open plugins/LuckTAB/config.yml and configure your ranks:

prefixes:
  - "owner;<gradient:#FF0000:#FF7700>[Owner] </gradient>;;&#FF7700"
  - "admin;&#FF5555[Admin] &7;;&#FF5555"
  - "mod;ᖳFF[Mod] &7;;ᖳFF"
  - "vip;&6[VIP] &7;;&6"
  - "default;&7[Player] &7;;&7"

Format: luckperms_group;prefix;suffix;namecolor

PartDescription
groupExact LuckPerms group name (case-sensitive)
prefixShown in tab list, nametag and chat
suffixLeave empty with ;; if not needed
namecolorColor applied to the player name

LuckPerms Setup

/lp user <player> group set owner
The group name must match exactly what you have in the prefixes list.

Migrate from LuckPerms

Already have prefixes in LuckPerms? Pull them into config:

/lucktab migrate from-luckperms

Or push your config prefixes into LuckPerms:

/lucktab migrate to-luckperms

⚙️ Configuration

# ── Prefix Source ─────────────────────────────
# lucktab   → use prefixes from this config (default)
# luckperms → use prefixes directly from LuckPerms meta
prefix-source: lucktab

# ── Ping Colors ───────────────────────────────
ping-colors:
  good: 50    # <= 50ms  → green
  medium: 100 # <= 100ms → yellow
              # >  100ms → red

# ── Tab-List ──────────────────────────────────
tab-list:
  show-only-same-world: false

# ── Chat Format ───────────────────────────────
useChatFormat: true
chatFormat: "%chatPrefix%%playerName% &8» &7%message%"

# ── Rank Prefixes ─────────────────────────────
# Format: "group;prefix;suffix;namecolor"
# Top = highest priority | suffix can be empty: "group;prefix;;color"
prefixes:
  - "owner;<gradient:#FF0000:#FF7700>[Owner] </gradient>;;&#FF7700"
  - "admin;&#FF5555[Admin] &7;;&#FF5555"
  - "mod;ᖳFF[Mod] &7;;ᖳFF"
  - "vip;&6[VIP] &7;;&6"
  - "default;&7[Player] &7;;&7"

# ── Header & Footer ───────────────────────────
# Per-world: add a section with the exact world name as key
# Falls back to "default" if no world-specific design is found
header-footer:
  enabled: true
  designs:
    default:
      header:
        - ""
        - "<gradient:#FF5733:#FFC300>   MyServer   </gradient>"
        - ""
      footer:
        - ""
        - "&fOnline: &b%server_online%&7/&b%server_max_players%  &7| &fPing: %lucktab_ping_colored%"
        - ""
    # world_nether:
    #   header:
    #     - "&4&l  The Nether  "
    #   footer:
    #     - "&fPing: %lucktab_ping_colored%"

# ── Scoreboard Sidebar ────────────────────────
scoreboard:
  enabled: true
  designs:
    default:
      title: "<gradient:#FF0000:#FF7700>MyServer</gradient>"
      lines:
        - ""
        - "&7Rank: %lucktab_rank%"
        - "&7Ping: %lucktab_ping_colored%"
        - ""
        - "&7World: &f%world%"
        - ""
        - "&7Online: &f%server_online%&8/&f%server_max_players%"
        - ""

Prefix Source

ValueDescription
lucktabPrefixes are read from config (default)
luckpermsPrefixes read directly from LuckPerms player meta — respects inheritance

Per-World Designs

Add a section with the exact world name under designs. LuckTAB falls back to default automatically.

World names are case-sensitive. Use /mv list or check your server folder to get the exact name.

Rank Priority

The order of entries in prefixes determines priority — top = highest. If a player has multiple groups, the highest entry wins.

Hex Colors

Use &#RRGGBB for any RGB color:

# Red prefix
"admin;&#FF0000[Admin] &7;;&#FF0000"

# Orange gradient color
"owner;[Owner] ;;&# FF7700"

# Custom purple
"mod;&#AA00FF[Mod] &7;;&#AA00FF"

Animated Gradients

Use <gradient:#RRGGBB:#RRGGBB>text</gradient>:

# Red to orange
"owner;<gradient:#FF0000:#FF7700>[Owner] </gradient>;;&#FF7700"

# Blue to purple
"admin;<gradient:#5555FF:#AA00FF>[Admin] </gradient>;;ᖳFF"

# Bold formatting preserved inside gradient
"owner;<gradient:#FF0000:#FF7700>&l[Owner]&r </gradient>;;&#FF7700"
Hex colors and gradients in scoreboard lines require Paper or Purpur. On Spigot, only legacy & codes work in scoreboard lines.

PlaceholderAPI

If PlaceholderAPI is installed, any third-party placeholder works everywhere:

lines:
  - "&7Balance: &f%vault_eco_balance%"
  - "&7Deaths: &f%statistic_deaths%"
  - "&7Level: &f%player_level%"
PlaceholderAPI placeholders are processed after LuckTAB built-in placeholders.

/lucktab migrate to-luckperms

Pushes all prefixes and suffixes from config.yml into LuckPerms group meta with automatic weight calculation:

Rank positionWeightPrefix priority
1st (top)200200
2nd190190
3rd180180
.........
Only LuckTAB-managed nodes are overwritten — other plugins' nodes are preserved.

/lucktab migrate from-luckperms

Pulls all group prefixes and suffixes from LuckPerms into config.yml. Groups without a prefix are skipped. Config is saved and rank map reloaded automatically.

/lucktab migrate from-luckperms
/lucktab reload

❓ FAQ

My rank prefix is not showing

  1. Make sure the group name in prefixes matches exactly the LuckPerms group name
  2. Check player group: /lp user <player> info
  3. Run /lucktab reload
  4. Top entry = highest priority — make sure owner is above default

The scoreboard is flickering

Make sure you are running Paper or Purpur. On Spigot, scoreboard lines with hex/gradient colors may flicker slightly due to a Minecraft client limitation.

Gradients look choppy or jump

This can happen if the text is very short (few characters = few color steps). Longer text produces smoother gradients. The animation uses sine-based interpolation which is smooth by design.

%lucktab_rank% shows raw gradient tag

LuckTAB cannot find a matching rank for the player. Make sure:

  • The player has a LuckPerms group assigned
  • That group exists in the prefixes list
  • The group name matches exactly (case-sensitive)

Does LuckTAB work on Folia?

No. LuckTAB uses Bukkit's scheduler which is not Folia-compatible. There are no plans to support Folia.

Does LuckTAB work on BungeeCord/Velocity?

LuckTAB is a per-server plugin. For network-wide player counts, use a PlaceholderAPI expansion that supports BungeeCord.
